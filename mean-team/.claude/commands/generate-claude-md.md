# /generate-claude-md

Generate a `CLAUDE.md` file at the project root that explains the project, lists all available agents, skills, commands, and shows how to invoke them.

## Context

$ARGUMENTS

## Instructions

You are the Agent Squad Architect running in Documentation Generation mode. Your job is to scan the entire `.claude/` ecosystem and produce a `CLAUDE.md` that serves as the instruction manual for both humans and AI agents working in this project.

### Process

#### 1. Scan Project Identity

Read project files to understand what this project is:

```
Read: package.json OR go.mod OR Cargo.toml OR pyproject.toml (whichever exists)
Read: README.md (if exists)
Glob: src/**/* OR lib/**/* OR app/**/* (first few files to understand the stack)
```

Extract: project name, language/framework, purpose.

#### 2. Discover All Components

Scan the `.claude/` directory for all components:

```
Glob: .claude/agents/*.md              (subagents)
Glob: .claude/skills/*/SKILL.md        (skills)
Glob: .claude/commands/*.md            (commands)
Glob: .claude/workflows/*.yaml         (orchestration pipelines)
Read: .claude/mcp.json                 (MCP servers, if exists)
```

For each agent found, read its frontmatter to extract: `name`, `description`, `model`, `skills`.

Exclude Architect infrastructure from the listing:
- **Commands:** `architect`, `create-agent`, `create-skill`, `create-el-capitan`, `el-capitan`, `compose-agents`, `insight`, `fleet-audit`, `invoke-el-capitan`, `generate-claude-md`
- **Skills:** `skill-creator`
- **Agents:** `AGENT_TEMPLATE.md`, `el-capitan-*.md`

#### 3. Discover Squads

For each workflow YAML in `.claude/workflows/`:
- Read the workflow definition
- Extract: squad name, phases, agents involved
- Map the pipeline flow

#### 4. Generate CLAUDE.md

Write `CLAUDE.md` at the project root with the following structure:

```markdown
# {Project Name}

{1-2 sentence description of what this project is and what stack it uses.}

## AI Agent Squad

This project has an AI agent squad that can be orchestrated to handle development tasks.

### Quick Start

```
# Run the full squad pipeline (auto-detects squad)
/el-capitan {your request}

# Preview what the squad will do (no file changes)
/el-capitan --dry-run {your request}

# With multiple squads, specify which one
/el-capitan --squad {squad-name} {your request}
```

### Available Agents

| Agent | Role | Model | Invoke |
|-------|------|-------|--------|
| {name} | {one-line from description} | {model} | `Task(subagent_type="{name}", prompt="...")` |
| ... | ... | ... | ... |

### Agent Details

#### {agent-name}
- **Role:** {description}
- **Skill:** {skill-name}
- **Invoke:** `Task(subagent_type="{agent-name}", prompt="your request here")`
- **Dry-run:** `Task(subagent_type="{agent-name}", prompt="your request --dry-run")`

{Repeat for each agent}

### Available Commands

| Command | Purpose |
|---------|---------|
| `/{command}` | {one-line description} |
| ... | ... |

### Squad Pipeline

```
/el-capitan {request}
       │  (auto-detects squad)
       ├─► Phase 0: {pm-agent} → PRD
       ├─► Phase 1: {agent} → {output}
       ├─► Phase 2: {agent} → {output}
       └─► Final Report
```

### Skills

| Skill | Used By | Purpose |
|-------|---------|---------|
| {skill-name} | {agent-name} | {one-line purpose} |

### Project Structure

```
.claude/
├── agents/          # Subagent definitions
├── skills/          # Skill playbooks (SKILL.md, FORMAT.md, VOICE.md, workflows/)
├── commands/        # Slash commands
├── workflows/       # Pipeline YAML definitions
├── agent-memory/    # Per-agent persistent memory
└── scripts/         # Scaffold scripts (init-skill.sh, init-agent.sh)
```
```

#### 5. Handle Edge Cases

- **No agents found:** Generate a minimal CLAUDE.md explaining the project only, with a note: "No agent squad configured yet. Run `/architect build {domain}` to create one."
- **No squads/workflows:** List agents individually without the pipeline section. Note: "Run `/create-el-capitan {squad-name}` to orchestrate these agents into a pipeline."
- **Multiple squads:** Create a section for each squad with its own pipeline diagram.
- **Existing CLAUDE.md:** Read it first. Preserve any user-written sections (look for comments like `<!-- custom -->` or sections not generated by this command). Replace only the AI Agent Squad sections.

#### 6. Report

After generating, output:

```
## CLAUDE.md Generated

- Project: {name}
- Agents listed: {count}
- Commands listed: {count}
- Skills listed: {count}
- Squads documented: {count}
- File: CLAUDE.md (project root)
```

### Quality Checklist

- [ ] Every agent has correct `Task(subagent_type="{name}", prompt="...")` invocation syntax
- [ ] Architect infrastructure is excluded from listings
- [ ] Squad pipeline diagram matches workflow YAML
- [ ] All agents reference their actual skill names
- [ ] Commands section only lists user-facing commands
- [ ] Dry-run instructions are included
- [ ] Project description is accurate (from package.json/README)
